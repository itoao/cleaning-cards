# UI/UX

## 画面一覧
- 起動画面
- カメラ/撮影
- 撮影ガイド
- カード表示(メイン)
- リザルト（完了一覧）
- 課金導線
- 設定/サポート

## 主要導線
- 起動 → 撮影 → ガイド → カード表示 → スワイプ完了 → 次カード
- 無料3枚目完了 → 課金導線

## 撮影/撮り直し導線
- 撮影前ガイド:
  - 光量・距離・角度のヒントを短く提示
  - 画角に部屋の主要エリアが入るよう促す
- 解析後の撮り直し:
  - 低信頼や判定不能の場合は「もう一度撮影」を提示
  - 失敗理由は責めずに簡潔に伝える
  - 例: 「少し暗かったかも もう一度撮影しよう」
- 任意の撮り直し:
  - ユーザーが「精度が低い」と感じた場合に手動で撮り直せる導線を用意
  - 例: カード画面に「撮り直す」ボタン/リンク
  - 進捗がある程度出た後でも再撮影できる
  - 完了スワイプと競合しないよう、再撮影はボタン/リンクで提供

## UI要件
- レイアウト:
  - フラッシュカードUIが主役
  - 背景に元写真をぼかして表示
  - 画面内の要素は最小限で判断負荷を下げる
- 入力/操作:
  - 完了操作はスワイプのみ
  - 1カード=1行動
  - 1文指示のみ表示
- フィードバック:
  - スワイプ完了時に紙吹雪
  - ハプティクスで完了感
  - アニメーションは穏やかで急かさない

## Visual Direction
- タイポグラフィ:
  - 読みやすさ重視
  - 太めの見出しと短い本文でメリハリ
- アイコン/イラスト:
  - シンプルで親しみやすい
  - 情報量は最小限

## App Icon
- 参考アイコンの特徴:
  - 角丸スクエアの淡いスカイブルー背景
  - 黄色のクラッカー(紙吹雪の筒)を斜めに配置
  - 紙吹雪はパステルの赤/ピンク/グリーン/イエロー
  - 柔らかい影で立体感を軽く付与
- トーン:
  - ポップでやさしい、子どもっぽくなりすぎない
  - 片付け完了の「軽い達成感」を表現

## UX要件
- 体験のゴール:
  - 考えずに始められる
  - 5〜15分で終わった感覚が得られる
- 失敗時の扱い:
  - 叱責や圧を避ける
  - 再開しやすい言葉と導線
- コピー方針:
  - 命令口調にしない
  - 句点(。)を使わない
  - 簡潔でやさしい言葉

## リザルト画面
- 最後に完了したタスク一覧を表示
- 取り直し前/取り直し後を含めた「最終完了タスク」を対象にする
- 進捗の達成感を優先し、詳細ログは最小限

## 進捗フィードバック
- スワイプ完了ごとに「進んでる感」を出す演出を入れる
- 例: 進捗バー/カウンタ、推定残り時間の減少、軽い達成エフェクト

## 終了見通しのUX
- 「いつ終わるか分からない」ストレスを減らす
- 正確さよりも“終わりが見える感”を優先
- 目安の残り時間レンジを表示し、スワイプごとに減る演出
- 再撮影・再解析があっても残り時間の増加は最小限に抑える
- カード内のタスク時間と区別するため、全体残りはカード外（上部）に固定表示

## Onboarding
- 初回体験:
  - いきなり「撮影 → カード表示 → スワイプ完了」の流れに入る
  - 説明は最小限、体験で理解させる
- 続ける導線:
  - 継続利用のタイミングでGoogle/Appleログインを提示
  - ログイン後に課金導線へ接続
  - 「課金」「購入」という言葉は使わない

## 禁止事項
- 複数タスクの同時提示
- チェックリスト型UI
- 高圧的・説教的な表現
- AI技術の強調
