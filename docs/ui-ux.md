# UI/UX

## 画面一覧
- 起動画面
- カメラ/撮影
- 撮影ガイド
- カード表示(メイン)
- 課金導線
- 設定/サポート

## 主要導線
- 起動 → 撮影 → ガイド → カード表示 → スワイプ完了 → 次カード
- 無料3枚目完了 → 課金導線

## UI要件
- レイアウト:
  - フラッシュカードUIが主役
  - 背景に元写真をぼかして表示
  - 画面内の要素は最小限で判断負荷を下げる
- 入力/操作:
  - 完了操作はスワイプのみ
  - 1カード=1行動
  - 1文指示のみ表示
- フィードバック:
  - スワイプ完了時に紙吹雪
  - ハプティクスで完了感
  - アニメーションは穏やかで急かさない

## Visual Direction
- トーン: ポップで明るい
- カラー:
  - 水色を基調
  - 高彩度のアクセント色を1〜2色
  - 背景は明るい中間色で写真を邪魔しない
- タイポグラフィ:
  - 読みやすさ重視
  - 太めの見出しと短い本文でメリハリ
- アイコン/イラスト:
  - シンプルで親しみやすい
  - 情報量は最小限

## App Icon
- 参考アイコンの特徴:
  - 角丸スクエアの淡いスカイブルー背景
  - 黄色のクラッカー(紙吹雪の筒)を斜めに配置
  - 紙吹雪はパステルの赤/ピンク/グリーン/イエロー
  - 柔らかい影で立体感を軽く付与
- トーン:
  - ポップでやさしい、子どもっぽくなりすぎない
  - 片付け完了の「軽い達成感」を表現
- 使用ルール:
  - 背景色はUI全体の基調(水色)と統一
  - 余白を確保して情報量を増やさない
  - 紙吹雪の数は少なめで可読性重視

## UX要件
- 体験のゴール:
  - 考えずに始められる
  - 5〜15分で終わった感覚が得られる
- 失敗時の扱い:
  - 叱責や圧を避ける
  - 再開しやすい言葉と導線
- コピー方針:
  - 命令口調にしない
  - 句点(。)を使わない
  - 簡潔でやさしい言葉

## 禁止事項
- 複数タスクの同時提示
- チェックリスト型UI
- 高圧的・説教的な表現
- AI技術の強調
